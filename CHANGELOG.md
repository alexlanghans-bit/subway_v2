## Release Procedure
1. Open a PR that updates docs and version bump.
2. Update `CHANGELOG.md` under **Unreleased** with changes.
3. When merging, move items into a new version section and stamp the date.
4. Tag the release in Git with the version (e.g., `v1.1.0`).
5. Distribute updated repo bundle if needed.

## Versioning Guidelines
- **MAJOR**: Breaking architecture/process changes, incompatible file moves/renames.
- **MINOR**: Backward-compatible additions (new region playbooks, KPIs, format specs).
- **PATCH**: Backward-compatible bug fixes (typos, small metric corrections).
